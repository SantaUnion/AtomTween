pixel-fitting animation 'x/y' vs 'left/top' using AtomTween.js

Square 1: AtomTween.to (pixelSquare1, 5, {delay:del+=0, left:800, top:100, ease:"ease-in-out"});
Square 2: AtomTween.to (pixelSquare2, 5, {delay:del+=0, x:800, y:170, ease:"ease-in-out"});

Both squares animate across the screen and 20px down.
Square 2 uses 'x' (ie translateX ) property to animate and is considerably smoother than Square 1 which uses the 'left' property.
